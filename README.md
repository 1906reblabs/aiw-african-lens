## How It Works

```
content/issues/aiw-026-NN-content.md   (Claude generates this — ~700 lines YAML)
                    ↓
              afi_parser.py             (parses YAML, converts markdown → HTML)
                    ↓
            templates/issue.j2          (Jinja2 template — design system)
                    ↓
        docs/weekly/aiw-026-NN.html     (final polished HTML — auto-committed)
```

On every push to `main`, GitHub Actions runs `python build.py all --force`, rebuilds every HTML file, and commits the output back to `docs/`. GitHub Pages serves `docs/` as the live site.

## To Publish a New Edition

### Step 1 — Generate content in Claude

Use this exact prompt:

```
Generate the content file for AIW-026-[NN].

Edition:  AIW-026-[NN]
Date:     [Day Month Year]
Window:   [Start] → [End]

Top 5 signals this week:
1. [Signal title + 1-sentence description]
2. [Signal title + 1-sentence description]
3. [Signal title + 1-sentence description]
4. [Signal title + 1-sentence description]
5. [Signal title + 1-sentence description]

Countries to spotlight: [Country 1], [Country 2]
SA comment days remaining: [N]

Output ONLY the complete YAML content file between --- markers,
following the schema of content/issues/aiw-026-11-content.md exactly.
No HTML. No explanation. No preamble.
```

Claude returns ~700 lines of structured YAML — nothing else.

### Step 2 — Save the file

Save Claude's output as `content/issues/aiw-026-NN-content.md`.

### Step 3 — Update the registry

Open `content/registry.yaml`. At the **top** of the `weekly:` list, add:

```yaml
- id: "AIW-026-NN"
  status: "new"
  badge: "gold"
  date: "[Day Month Year]"
  window: "[Start] → [End]"
  window_short: "[Start] → [End]"
  title: "[Edition title]"
  subtitle: "[Edition subtitle]"
  summary: >
    [2–3 sentence summary for the featured card]
  themes_line: "[Theme 1] · [Theme 2] · [Theme 3] · [Theme 4] · [Theme 5]"
  developments: 10
  file: "aiw-026-NN.html"
```

Change the previous entry from `status: "new"` → `status: "available"` and `badge: "gold"` → `badge: "teal"`.

Increment `stats.weekly_total` by 1.

### Step 4 — Push to GitHub

```bash
git add content/
git commit -m "content: add AIW-026-NN — [Edition title]"
git push
```

GitHub Actions triggers automatically. Within ~60 seconds:
- `docs/weekly/aiw-026-NN.html` is built and committed
- `docs/index.html` is rebuilt
- `docs/weekly/index.html` is rebuilt
- The live site is updated

## Local Build (optional)

```bash
pip install -r requirements.txt

# Build a single issue
python build.py issue content/issues/aiw-026-12-content.md

# Rebuild both index pages
python build.py index

# Build everything
python build.py all --force
```

Output goes to `docs/` by default.

## Content Schema Reference

The YAML schema for each issue is defined by `content/issues/aiw-026-11-content.md`.  
Key top-level fields:

| Field | Type | Description |
|---|---|---|
| `edition_id` | string | e.g. `AIW-026-12` |
| `date` | string | e.g. `25 May 2026` |
| `window_start` / `window_end` | string | Coverage window |
| `title` / `title_sub` | string | Edition headline |
| `deck` | string | Masthead summary (1 paragraph) |
| `developments_count` | int | Number of ranked developments |
| `nations_covered` | int | Nations covered this issue |
| `sa_comment_days_remaining` | int | SA policy countdown |
| `opening_frame` | markdown | §1 editorial paragraph |
| `signals` | list[5] | Top 5 signals with body + 3-tier analysis |
| `layers` | list[5] | Energy/Chips/Infra/Models/Apps analysis |
| `policy_status_pills` | list | Status badges for policy section |
| `policy_head` | string | Policy watch section headline |
| `policy_body_1` / `policy_body_2` | markdown | Policy prose blocks |
| `convergence_rows` | list | Policy convergence table rows |
| `research_african` | dict | African-led research card |
| `research_global` | dict | Global/institutional research card |
| `fragility_items` | list[2-3] | Fragility assessments with mitigations |
| `contrarian_items` | list[1-2] | Contrarian bet assessments |
| `spotlights` | list[1-3] | Country spotlights |
| `forward_items` | list[6] | Forward look calendar |
| `editorial_note` | string | Bottom editorial note |
